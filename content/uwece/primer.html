Primer for Understanding the Unix Hierarchy

This page is a modern companion to a classic 1986 USENET↗ post describing "Unix wizardry" as a progression of tools, habits, and confidence.

Primary source: the original USENET posting (Oct 1986). See the post in UNIX.html.

Hierarchy, annotated

Beginner

  • terminal - Frequently, access to computer systems were provided as dumb-terminals connected to mainframe computers. A dumb-terminal was essentially a CRT display of 80 characters wide and 24 (or 25) lines deep, with the most un-ergonomic keyboard imaginable. There was no mouse, no windows, no task bar and definitely no color.
  • vi - A visual editor. All editing function of the editor can be performed by keyboard. Your hands do not need to leave the keyboard. Same can be said about emacs. The same cannot be said for notepad/VisualStudio/Eclipse/etc.

Novice

  • ls - A command to list files.
  • C - A computer programming language invented by Dennis Ritchie.
  • rm - A command to remove files.

User

  • nroff - A typesetting tool. For an example, examine the source of any Unix man page:
    • $ gzip -dc /usr/share/man/man1/grep.1.gz | less

    Side question -- what do the numbers mean? Manual pages are organized in sections:

    • 1 - User commands (ls, grep, awk, etc.)
    • 2 - System calls (open, read, close, etc.)
    • 3 - Library calls (libc, libm, etc.)
    • 4 - Special files (Files found in /dev, etc.)
    • 5 - File formats and conventions (/etc/passwd, etc)
    • 6 - Games
    • 7 - Miscellaneous (conventions, macros, protocols)
    • 8 - System administration commands (ifconfig, etc)
    • 9 - (non standard section) Kernel routines and device driver information
  • pascal - A structured programming language used heavily for teaching modular programming; later replaced by variants and then (more recently) by Java for instruction in many curricula.
  • dbx - A type of debugger.
  • ...how to read the news... - Reading the news referred to the posting system called USENET

Knowledgeable User

  • tbl, eqn - Programs that can process textual data (see their man pages).
  • grep - Finds patterns in files.
  • mv - Moves files. In Unix: everything is a file.
  • sed - A stream editor for pattern matching and replacement in pipelines.
  • make - A tool to build things from rules (compile, generate, prepare, etc.).

Expert

  • ex - A line editor related to vi; useful when you cannot rely on full-screen terminal behavior.
  • csh - Historically popular for interactive use; notorious for risky scripting.
Never write scripts in csh.
  • &&, || - Boolean operators in the shell:
$ foo && bar

Hacker

  • sed & awk - Text processing power tools; awk is named after Aho, Weinberger, and Kernighan.
  • cat > - Dictating code directly to a file without an editor (the fountain-pen crossword puzzle tactic).
  • adb - Another debugger.
  • environment variables - Variables inherited by child processes (vs. shell variables that are not). Example:
$ x=42 $ bash -c 'echo $x' # prints nothing $ export y=99 $ bash -c 'echo $y' # prints 99
  • Bourne shell (sh) - Stephen Bourne (Bell Labs) wrote sh for scripting and portability; descendants include POSIX sh and bash.
  • ...from the net... - Acquiring patches via FTP or ordered tapes; later, downloading bug fixes and updates from the network.

Guru

  • m4 - A general-purpose macro processor operating on plain text; powerful and dangerous via unrestricted expansion.
  • lex - A generator for lexical analyzers from regular-expression token rules; traditionally paired with yacc.
  • James Woods/Henry Spencer - Associated with extended grep/egrep implementations.

Wizard

  • posts changes to Unix utilities to the net - Public patch sharing, often via USENET.
  • Dennis, Bill, and Ken - Dennis Ritchie↗, Bill Joy↗ (BSD), and Ken Thompson↗ (Unix origin).
